# 기능 요구사항 명세서

## 1. 앱 기본 정보

- 앱 이름: 미정
- 플랫폼: Android
- 개발 방식: Flutter
- 배포 방식: APK 직접 전달
- 음원 방식: 서버 URL 스트리밍

## 2. 기능 요구사항

### FR-001. 곡 목록 불러오기

- 앱은 실행 시 서버의 `songs.json`을 요청한다.
- 요청 성공 시 곡 목록을 화면에 표시한다.
- 요청 실패 시 오류 메시지와 다시 시도 버튼을 표시한다.

### FR-002. 곡 목록 표시

- 각 곡은 제목, 아티스트, 커버 이미지를 표시한다.
- 커버 이미지가 없거나 로딩 실패 시 기본 이미지를 표시한다.
- 현재 재생 중인 곡은 목록에서 시각적으로 구분한다.

### FR-003. 곡 재생

- 사용자가 곡을 누르면 해당 곡을 스트리밍으로 재생한다.
- 재생 중 다른 곡을 누르면 기존 재생을 중지하고 새 곡을 재생한다.
- 재생 실패 시 사용자에게 오류를 표시한다.

### FR-004. 플레이어 컨트롤

- 앱은 다음 컨트롤을 제공한다.
  - 재생
  - 일시정지
  - 다음곡
  - 이전곡
  - 재생 위치 이동

### FR-005. 재생 상태 표시

- 현재 곡 제목을 표시한다.
- 현재 곡 커버 이미지를 표시한다.
- 재생 중/일시정지 상태를 표시한다.
- 현재 재생 위치와 전체 길이를 표시한다.

### FR-006. 백그라운드 재생

- 앱이 백그라운드로 이동해도 재생을 유지한다.
- Android 알림창에 미디어 컨트롤을 표시한다.
- 잠금화면 미디어 컨트롤을 지원한다.

### FR-007. 다음곡/이전곡

- 다음곡 버튼을 누르면 현재 목록 기준 다음 곡으로 이동한다.
- 이전곡 버튼을 누르면 현재 목록 기준 이전 곡으로 이동한다.
- 마지막 곡에서 다음곡을 누르면 첫 곡으로 이동한다.
- 첫 곡에서 이전곡을 누르면 마지막 곡으로 이동한다.

### FR-008. 네트워크 오류 처리

- 곡 목록 요청 실패 시 다시 시도할 수 있어야 한다.
- 음원 재생 실패 시 해당 곡을 재생할 수 없다는 안내를 표시한다.
- 서버 응답 형식이 잘못된 경우 앱이 종료되지 않아야 한다.

## 3. 비기능 요구사항

### NFR-001. 사용성

- 주요 버튼은 아이가 누르기 쉬운 크기로 만든다.
- 첫 화면에서 곡 목록과 재생 상태를 바로 볼 수 있어야 한다.
- 설정성 기능은 1차 버전에서 노출하지 않는다.

### NFR-002. 안정성

- 네트워크가 불안정해도 앱이 강제 종료되지 않아야 한다.
- 잘못된 곡 데이터가 일부 있어도 가능한 곡은 표시/재생해야 한다.

### NFR-003. 성능

- 곡 목록은 100곡까지 무리 없이 표시할 수 있어야 한다.
- 이미지 로딩으로 스크롤이 심하게 끊기지 않아야 한다.

### NFR-004. 보안

- 서버 URL은 HTTPS를 사용한다.
- 1차 버전에서는 별도 로그인 없이 공개 URL 접근을 전제로 한다.
- 추후 필요 시 토큰 기반 접근 제한을 추가할 수 있어야 한다.

## 4. 화면 요구사항

### SCR-001. 홈 화면

- 곡 목록을 표시한다.
- 하단에 미니 플레이어를 표시한다.
- 로딩, 오류, 빈 목록 상태를 구분해서 표시한다.

### SCR-002. 전체 플레이어 화면

- 큰 커버 이미지
- 곡 제목/아티스트
- 재생 위치 슬라이더
- 이전/재생/다음 버튼

## 5. 데이터 모델

앱 내부의 곡 모델은 서버 JSON과 거의 동일하게 유지한다.

```dart
class Song {
  final String id;
  final String title;
  final String artist;
  final Uri audioUrl;
  final Uri? coverUrl;
  final Duration? duration;
}
```

## 6. 기술 후보

- `just_audio`: 스트리밍 재생
- `audio_service`: 백그라운드 재생 및 Android 미디어 알림
- `audio_session`: Android/iOS 오디오 세션 설정
- `http`: JSON 요청
- `cached_network_image`: 커버 이미지 캐싱
- `shared_preferences`: 마지막 재생 곡 등 간단한 로컬 상태
