# 조카용 음악 플레이어 제품 기획서

## 1. 제품 요약

조카가 직접 누르고 들을 수 있는 Android 전용 음악 플레이어 앱을 만든다.

앱은 Flutter로 개발하며 Google Play Store에 배포하지 않고 APK 파일로 직접 전달한다. 음원 파일은 앱에 포함하지 않고, 서버에 있는 곡 목록 JSON과 음원 URL을 불러와 스트리밍으로 재생한다.

## 2. 사용자

### 주 사용자

- 어린 조카
- 복잡한 메뉴보다 큰 버튼과 단순한 화면이 필요한 사용자
- 곡 제목을 정확히 읽지 못해도 커버 이미지와 위치로 곡을 선택할 수 있어야 함

### 보조 사용자

- 보호자
- 서버에 음원을 올리고 곡 목록을 관리하는 사람
- APK 파일을 설치하거나 업데이트해주는 사람

## 3. 제품 목표

- 아이가 실수 없이 노래를 재생할 수 있게 한다.
- 보호자가 앱을 다시 빌드하지 않아도 서버에서 곡을 추가/삭제할 수 있게 한다.
- Android에서 일반 음악 앱처럼 백그라운드 재생과 알림 컨트롤을 제공한다.
- 가족용 개인 앱으로 빠르게 전달 가능한 APK 산출물을 만든다.

## 4. 핵심 가치

- 단순함: 화면과 동작을 최소화한다.
- 안정성: 재생이 끊기거나 앱 상태가 꼬이지 않게 한다.
- 관리 편의성: 곡 추가/삭제는 서버 JSON 수정으로 처리한다.
- 아이 친화성: 터치 영역을 크게 만들고 위험한 설정은 노출하지 않는다.
- 일관된 음악앱 감성: `DESIGN-spotify.md`의 다크 테마, pill/circle 버튼, 하단 플레이어 구조를 따른다.

## 5. 1차 버전 범위

1차 버전은 음악 재생의 핵심 흐름에 집중한다.

- 서버 곡 목록 불러오기
- 곡 리스트 표시
- 선택한 곡 스트리밍 재생
- 재생 / 일시정지
- 다음곡 / 이전곡
- 현재 재생 중인 곡 표시
- 백그라운드 재생
- Android 알림창/잠금화면 미디어 컨트롤
- APK 빌드

## 6. 1차 버전에서 제외할 기능

아래 기능은 1차 버전 이후로 미룬다.

- 회원가입/로그인
- 관리자 웹 페이지
- 오프라인 다운로드
- 즐겨찾기
- 재생 기록
- 카테고리 편집
- 보호자 잠금
- 수면 타이머
- 서버 파일 업로드 기능

## 7. 주요 사용 시나리오

### 시나리오 1: 아이가 노래를 고른다

1. 앱을 실행한다.
2. 서버에서 곡 목록을 불러온다.
3. 아이가 곡 커버 또는 제목을 누른다.
4. 선택한 곡이 재생된다.

### 시나리오 2: 화면을 꺼도 노래가 계속 나온다

1. 곡을 재생한다.
2. 화면을 끄거나 앱을 백그라운드로 보낸다.
3. 음악은 계속 재생된다.
4. 알림창 또는 잠금화면에서 일시정지/다음곡을 누를 수 있다.

### 시나리오 3: 보호자가 곡을 추가한다

1. 서버에 음원 파일과 커버 이미지를 업로드한다.
2. `songs.json`에 곡 정보를 추가한다.
3. 앱을 다시 실행하거나 새로고침하면 곡 목록에 반영된다.

## 8. 성공 기준

- 실제 Android 기기에서 APK 설치가 가능해야 한다.
- 서버 JSON을 기준으로 곡 목록이 표시되어야 한다.
- 최소 10곡 이상의 원격 음원을 연속 재생할 수 있어야 한다.
- 앱을 백그라운드로 보낸 뒤에도 재생이 유지되어야 한다.
- 알림창에서 재생/일시정지/다음곡/이전곡이 동작해야 한다.

## 9. 법적/운영 주의사항

- 앱은 합법적으로 사용할 수 있는 음원만 연결한다.
- 저작권이 있는 음원을 무단 다운로드하거나 배포하지 않는다.
- 서버 URL이 공개되어 있다면 링크를 아는 사람이 접근할 수 있다.
- 필요하면 서버 접근 토큰 또는 비공개 스토리지 방식을 추후 검토한다.
