# 조카용 음악 플레이어 앱 기획안

## 1. 개요

조카가 쉽고 안전하게 노래를 들을 수 있는 Android 전용 음악 플레이어 앱을 만든다.

앱은 Flutter로 개발하고, 앱스토어/플레이스토어에 등록하지 않고 APK 파일로 직접 전달한다. 음원 파일은 앱 내부에 포함하지 않고, 서버에 올라와 있는 음원 목록과 음원 URL을 불러와 스트리밍 방식으로 재생한다.

## 2. 목표

- 아이가 직접 사용할 수 있는 단순하고 큰 UI 제공
- 서버의 음원 목록을 앱에서 자동으로 불러오기
- 곡 선택, 재생, 일시정지, 다음곡, 이전곡 지원
- Android 백그라운드 재생 지원
- 잠금화면 및 알림창의 미디어 컨트롤 지원
- APK 파일로 가족에게 직접 전달 가능

## 3. 배포 방식

- 대상 플랫폼: Android
- 배포 방식: APK 직접 전달
- 스토어 등록: 하지 않음
- 설치 방식:
  - APK 파일 전달
  - Android 기기에서 "알 수 없는 앱 설치 허용" 후 설치

## 4. 음원 관리 방식

음원은 앱에 직접 포함하지 않고 서버에서 관리한다.

서버에는 다음 파일들을 둔다.

- 음원 파일: `.mp3`, `.m4a` 등
- 커버 이미지: `.jpg`, `.png`, `.webp` 등
- 곡 목록 JSON: `songs.json`

앱은 실행 시 `songs.json`을 불러와 곡 목록을 구성한다.

### 예시 JSON

```json
[
  {
    "id": "song-001",
    "title": "노래 제목",
    "artist": "아티스트",
    "url": "https://example.com/audio/song-001.mp3",
    "coverUrl": "https://example.com/covers/song-001.png",
    "duration": 120
  }
]
```

## 5. 주요 기능

### 5.1 기본 재생

- 곡 목록 표시
- 곡 선택 재생
- 재생 / 일시정지
- 다음곡 / 이전곡
- 재생 위치 표시
- 곡 길이 표시

### 5.2 음악 플레이어 기능

- 반복 재생
- 전체 반복
- 셔플 재생
- 현재 재생 중인 곡 강조 표시
- 재생 중인 곡 커버 이미지 표시

### 5.3 백그라운드 재생

- 앱을 닫거나 화면을 꺼도 음악 재생 유지
- Android 알림창에서 재생 / 일시정지 / 다음곡 / 이전곡 조작
- 잠금화면 미디어 컨트롤 지원

### 5.4 네트워크 처리

- 서버 곡 목록 불러오기
- 로딩 상태 표시
- 네트워크 오류 시 안내 표시
- 음원 재생 실패 시 다음 곡으로 넘어가거나 재시도

## 6. 추후 기능 후보

- 즐겨찾기
- 최근 들은 곡
- 카테고리 또는 앨범 구분
- 선택한 곡 오프라인 저장
- 보호자 설정 화면
- 앱 실행 시 자동으로 마지막 곡 이어듣기
- 취침 타이머
- 볼륨 제한 또는 큰 소리 방지

## 7. 기술 스택

- Framework: Flutter
- Language: Dart
- Audio playback: `just_audio`
- Background playback: `audio_service`
- Network: `http` 또는 `dio`
- Local storage: `shared_preferences`
- Optional cache: `flutter_cache_manager`

## 8. 앱 화면 구성

### 8.1 메인 화면

- 상단: 앱 이름
- 중앙: 곡 목록
- 하단: 미니 플레이어

### 8.2 플레이어 화면

- 커버 이미지
- 곡 제목
- 아티스트
- 재생 위치 슬라이더
- 이전곡 / 재생 / 다음곡 버튼
- 반복 / 셔플 버튼

### 8.3 오류 화면

- 서버 연결 실패 안내
- 다시 시도 버튼

## 9. 서버 요구사항

- HTTPS 지원
- `songs.json` 외부 접근 가능
- 음원 파일 URL 직접 접근 가능
- 커버 이미지 URL 직접 접근 가능
- CORS는 앱에서는 큰 문제가 되지 않지만, 추후 웹 관리 페이지가 생길 경우 고려

## 10. 저작권 및 주의사항

- 앱에는 합법적으로 사용할 수 있는 음원만 연결한다.
- 저작권이 있는 음원을 무단 다운로드하거나 배포하지 않는다.
- 가족 내 개인 사용이라도 저작권 및 서비스 약관을 확인한다.
- 서버 URL이 공개되어 있다면 링크를 아는 사람이 접근할 수 있으므로 필요 시 접근 제어를 고려한다.

- 스트리밍 재생
- 재생 / 일시정지
- 다음곡 / 이전곡
